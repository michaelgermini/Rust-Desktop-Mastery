# Chapitre 1 : Pourquoi Rust pour le Logiciel Moderne

Ce chapitre explore les raisons fondamentales qui font de Rust le choix idéal pour les applications desktop modernes.

   - Instabilité de l'écosystème npm
   - Dépendance aux services SaaS

2. **[Performance native](./02-performance-native.md)**
   - Zero-cost abstractions
   - Comparaison avec JavaScript/JVM
   - Impact sur l'expérience utilisateur

3. **[Sécurité mémoire](./03-securite-memoire.md)**
   - Ownership et borrowing
   - Bugs mémoire évités à la compilation
   - Null safety avec Option

4. **[Binaire unique](./04-binaire-unique.md)**
   - Déploiement simplifié
   - Pas de DLL hell
   - Portabilité et distribution

5. **[Offline-first](./05-offline-first.md)**
   - Architecture locale par défaut
   - Synchronisation optionnelle
   - Résilience réseau

6. **[Souveraineté des données](./06-souverainete-donnees.md)**
   - Données sous contrôle utilisateur
   - Chiffrement local
   - Conformité RGPD simplifiée
